## Root Cause Analysis (Source-Confirmed)

### B6a — P0 BLOCKER: Chart Container Has No Height (WO-B0c Regression)

**File:** `src/components/wellness-journey/SessionVitalsTrendChart.tsx` — **line 360**

```tsx
// CURRENT (BROKEN):
<div className={hideHeader ? 'w-full flex flex-col' : 'w-full bg-slate-900/60 ...'}>

// WO-B0c specified this fix but it was NOT APPLIED:
<div className={hideHeader ? 'w-full h-full flex flex-col' : 'w-full bg-slate-900/60 ...'}>
```

**Why it breaks:** When the cockpit renders the chart with `hideHeader={true}`, the outer `<div>` is `flex-col` with NO height. The inner chart area at line 406 is `h-full min-h-[220px]` — but `h-full` resolves relative to its flex parent. Since the parent has no height, `h-full = 0px`. `<ResponsiveContainer height="100%">` inherits `0px` → renders at zero height → blank canvas.

**Fix:** Add `h-full` to line 360. One character change. The WO-B0c plan specified this but the commit missed it.

```tsx
// APPLY THIS CHANGE — line 360 of SessionVitalsTrendChart.tsx:
<div className={hideHeader ? 'w-full h-full flex flex-col' : 'w-full bg-slate-900/60 backdrop-blur-xl border border-slate-700/50 rounded-2xl p-6 shadow-xl flex flex-col gap-4'}>
```

> ⚠️ This is a one-line fix. Do NOT refactor anything else. Surgical only.

### B6b — P1: Initial Dose Is NOT in DosingSessionPhase.eventLog

**File:** `src/components/wellness-journey/DosingSessionPhase.tsx`

The Live Session Timeline shows `T+00:00 · Initial Dose` because `LiveSessionTimeline` has its own internal event listener (`ppn:session-event`) and state. However, `DosingSessionPhase.eventLog` (which flows to the chart's `events` prop) is NOT populated with the initial dose entry.

The `ppn:dose-registered` event (dispatched by SessionPrepView when dosing starts) is NOT being listened for in `DosingSessionPhase` to append an entry to `eventLog`.

**Fix:** In the `ppn:session-event` listener inside `DosingSessionPhase`, also listen for `ppn:dose-registered` and push a `SessionEventPin` entry to `eventLog`:

```typescript
const handleDoseRegistered = (e: Event) => {
    const { type, label, elapsedSec } = (e as CustomEvent).detail;
    setEventLog(prev => [
        ...prev,
        {
            id: `dose-${Date.now()}`,
            type: type ?? 'dose_admin',
            label: label ?? 'Initial Dose',
            elapsedSec: elapsedSec ?? 0,
        }
    ]);
};
window.addEventListener('ppn:dose-registered', handleDoseRegistered);
// return () => window.removeEventListener('ppn:dose-registered', handleDoseRegistered);
```

This is where `eventLog` is built — look for the existing `ppn:session-event` listener pattern in `DosingSessionPhase.tsx` and add `ppn:dose-registered` alongside it.

## Key Architecture Facts

### Event Bus
- `ppn:session-event` → `LiveSessionTimeline` internal listener (for timeline display) AND `DosingSessionPhase.eventLog` state (for chart events prop)
- `ppn:dose-registered` → currently only received by LiveSessionTimeline, NOT by DosingSessionPhase

### Vitals Chart Data Flow
```
DosingSessionPhase.updateLog (SessionUpdateEntry[])
  → vitalsChartData (VitalsSnapshot[])           [useMemo line 774]
    → SessionCockpitView props.data
      → SessionVitalsTrendChart props.data

DosingSessionPhase.eventLog (SessionEventPin[])
  → SessionCockpitView props.events (inferred — verify this thread)
    → SessionVitalsTrendChart props.events
```

### localStorage Keys (Session State)
- `ppn_session_mode_<sessionId>` — `'pre'` | `'live'` | `'post'`
- `ppn_session_start_<sessionId>` — Unix ms timestamp of Start tap
- `ppn_dosing_protocol` — dosing protocol form data including baseline vitals
- `ppn_patient_medications_names` — medication display names (used by contraindication engine)

### Contexts
- `ActivePatientContext` — carries `activePatientUuid` + `activeSessionId` in URL hash
- `ActiveSessionsContext` — tracks running sessions for header chips
