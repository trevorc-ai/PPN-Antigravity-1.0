## Summary

The current Discharge Summary export in `src/services/dischargeSummary.ts` generates and downloads a **plain `.txt` file** — not a real PDF. For a clinical document of this importance, this is inappropriate. It needs to be upgraded to a properly formatted, branded PDF using a lightweight browser-native PDF library.

## Current State (Problem)

- File: `src/services/dischargeSummary.ts`, `downloadDischargeSummary()` (line 104)
- Output: `text/plain` Blob → downloads as `.txt`
- Format: Monospaced plain text, `\n`-joined lines, no layout, no branding
- No header/footer, no clinic logo, no page break handling
- Clinical narrative fields are free-text placeholder strings

## Proposed Solution

Use **`jspdf`** (already a common transitive dependency in Vite/React stacks) or **browser `window.print()`** with a hidden styled HTML template. Prefer `jspdf` for a direct download without a print dialog.

### Option A — `jspdf` (Recommended)
```bash
npm install jspdf
```

Build a `generateDischargeSummaryPDF(data: DischargeSummaryData)` function that:
1. Creates a new `jsPDF` document (A4, portrait)
2. Renders structured sections with correct typography hierarchy
3. Downloads as `.pdf` (not `.txt`)

### Option B — Hidden HTML template + `window.print()`
- Inject a hidden `<div>` with `@media print` styling
- Call `window.print()` 
- No library needed, but opens print dialog (worse UX for practitioners)

**Recommendation: Option A.**

## PDF Document Spec

The PDF should include these sections in order:

### Header
- PPN logo or clinic name (left)
- "CLINICAL DISCHARGE SUMMARY" title (right)
- Document date + Patient ID (right-aligned)
- Horizontal rule

### Section 1 — Patient & Treatment Summary
| Field | Source |
|---|---|
| Patient ID | `data.patientId` |
| Site | `data.siteId` (from auth) |
| Clinician ID | `data.clinicianId` |
| Treatment Dates | `data.treatmentStart` → `data.treatmentEnd` |
| Dosing Sessions | `data.dosingSessionsCount` |
| Integration Sessions | `data.integrationSessionsCount` |
| Substance & Dose | `data.substanceName`, `data.substanceDose` |
| Protocol | `data.protocolName` |
| Diagnosis | `data.diagnosis` |

### Section 2 — Outcome Metrics
- PHQ-9, GAD-7, CAPS-5: Baseline → Final → Δ (only if not null)
- MEQ-30 Peak score
- Response Achieved / Remission Achieved badges (color-coded: green YES, red NO)

### Section 3 — Safety Summary
- Adverse events logged
- Grade 3+ events
- Chemical rescue used

### Section 4 — Ongoing Care Plan
- Ongoing Vulnerabilities
- Referral (name or "Pending")
- Follow-up schedule
- Emergency plan
- Self-care instructions

### Section 5 — Clinician Sign-Off
- Clinician statement text block
- Clinician ID + Date printed
- Signature line: `_______________` with "Signature" label
- Disclaimer: *"Auto-generated by PPN. Clinician review required before patient distribution."*
