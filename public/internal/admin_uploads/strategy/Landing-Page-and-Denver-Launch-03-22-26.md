from typing import Optional

class LeadCaptureSchema(BaseModel):

    institution_name: str

    # The Ruthless Advisor Validation: Block free emails

    def block_free_emails(cls, v):
        free_domains = ['gmail.com', 'yahoo.com', 'hotmail.com', 'outlook.com']
        domain = v.split('@')[1].lower()
        if domain in free_domains:
            raise ValueError('Please use a professional or institutional email address.')
        return v

from app.schemas import LeadCaptureSchema
from app.services import crm_service, email_service

app = FastAPI(title="PPN Public Lead Capture API")

@app.post("/api/v1/leads/capture")
async def capture_institutional_lead(lead: LeadCaptureSchema, background_tasks: BackgroundTasks):
    try:
        # 1. Save to Database / CRM
        lead_record = crm_service.create_lead(lead)
        
        # 2. Trigger Background Task (Don't make the user wait)
        if lead.lead_source == "audit_pack_download":

\-----Detailed Section-by-Section Build Focus

The tutorial is broken into focused modules, ensuring a deep understanding of the techniques applied in each stage.1. Header (Banner and Navbar) and Hero Section

This foundational module defines the page's visual identity, navigation, and initial user impression.

* **Design & Layout Emphasis:** The core focus is on **responsive design** from the first line of code. We utilize Tailwind's full suite of responsive breakpoints (`sm`, `md`, `lg`) and master the application of `flexbox` and `grid` **utility classes** for optimal, adaptable layout.

* **Preparation for Animation:** Careful **positioning of decorative images** (using `absolute` and `relative` positioning) is meticulously executed in this stage. This preparation is essential for seamlessly implementing sophisticated Parallax or other interactive effects in later development modules.

2\. Logo Ticker

This section is dedicated to creating a visually engaging showcase of client or partner logos.

* **Core Animation Technique:** The primary technical challenge is achieving a **seamless, continuous infinite scroll effect**. This is accomplished by strategically utilizing a **linear gradient mask** applied to the edges of the ticker container. The mask, implemented using custom CSS or Tailwind's `bg-gradient-to-r` combined with a `mask-image` property, softly fades the logos in and out as they enter and exit the view. This technique eliminates hard visual breaks and smoothly integrates the repeated element loop, creating the illusion of perpetual motion.

3\. Product Showcase

The Product Showcase is where the project highlights key features, work samples, or case studies with visual depth and interactivity.

* **Styling Consistency:** The build heavily emphasizes creating and reusing consistent **utility styles** for all recurring elements, such as titles and descriptions. This involves defining specific utility classes for font sizes, weights, line heights, and letter-spacing to ensure a unified and professional typographic hierarchy across the page.  
* **Interactive Depth:** **Decorative images** are introduced purely for visual interest and are strategically placed using precise **absolute positioning**. These elements are designated as the specific targets for **Framer Motion's Parallax effects**. The resulting subtle shifting of these elements relative to the rest of the content as the user scrolls adds significant visual sophistication and a highly premium feel to the section.

4\. Pricing Section (Responsive and Dynamic)

This final, complex section focuses on clean, efficient data display and sophisticated conditional styling.

* **Dynamic Generation:** The section is architected for efficiency by utilizing efficient **data mapping** over a simple, structured JavaScript array of pricing tiers. This approach generates the individual pricing cards dynamically, drastically reducing repetitive JSX code and improving maintainability.  
* **Styling System:** A detailed and defined set of **utility classes** are consistently applied to maintain consistent padding, border styles, typography, and hover effects across all pricing cards.  
* **Conditional Variations (Advanced Technique):** The most sophisticated aspect here is the programmatic handling of card variations, such as the visually striking "inverse" (e.g., dark background with light text) and the highlighted "popular" (e.g., a card with a distinct border or shadow) card. This is managed programmatically within React logic and robustly styled using the **Tailwind Merge** library. Tailwind Merge ensures that conditional classes (e.g., a dark background class) are applied correctly without conflicting with or being overridden by the base utility styles, resulting in stable and predictable final styling.
