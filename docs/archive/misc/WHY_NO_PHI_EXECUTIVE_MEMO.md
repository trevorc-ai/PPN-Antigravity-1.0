### **How We Track Subjects Without PHI**

**The Challenge:** Practitioners need to track patients across multiple sessions

**The Solution:** Hashed subject IDs (within-site only)

**How It Works:**
1. Practitioner creates subject: `subject_id = hash(clinic_id + patient_initials + DOB_year)`
2. Example: `hash(clinic_123 + "JS" + 1982) = "a3f5b9c2"`
3. Practitioner uses `subject_id = "a3f5b9c2"` for all sessions
4. **Result:** Practitioner can track subject, but we can't identify patient

**What We Store:**
- ‚úÖ `subject_id`: "a3f5b9c2" (meaningless to us)
- ‚úÖ `age_range`: "36-45" (not exact age)
- ‚úÖ `indication_id`: 3 (MDD, from controlled vocabulary)
- ‚úÖ `substance_id`: 1 (Ketamine, from controlled vocabulary)

**What We DON'T Store:**
- ‚ùå Patient name
- ‚ùå Patient DOB
- ‚ùå Patient address
- ‚ùå Any free-text notes

**The Result:**
- Practitioner can track patient across sessions
- We can aggregate data for benchmarking
- **Zero PHI = Zero HIPAA risk**
