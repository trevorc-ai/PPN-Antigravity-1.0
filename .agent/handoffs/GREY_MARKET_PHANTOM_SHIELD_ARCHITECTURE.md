## ðŸ”§ TECHNICAL ARCHITECTURE

### Database Schema (New Tables)

**For WO-059 (Potency Normalizer):**
```sql
CREATE TABLE substance_batches (
    batch_id UUID PRIMARY KEY,
    practitioner_id UUID REFERENCES auth.users(id),
    substance_type VARCHAR(50),
    strain_name VARCHAR(50),
    potency_coefficient DECIMAL(3, 2) DEFAULT 1.0,
    reagent_test_image_url TEXT,
    has_fentanyl_strip_test BOOLEAN
);

CREATE TABLE ref_substance_strains (
    strain_id UUID PRIMARY KEY,
    substance_type VARCHAR(50),
    strain_name VARCHAR(100),
    default_potency_coefficient DECIMAL(3, 2) DEFAULT 1.0
);
```

**For WO-060 (Crisis Logger):**
```sql
CREATE TABLE session_interventions (
    intervention_id UUID PRIMARY KEY,
    session_id UUID REFERENCES log_sessions(session_id),
    event_type VARCHAR(50),
    logged_at_utc TIMESTAMP WITH TIME ZONE,
    seconds_since_ingestion INTEGER,
    heart_rate_bpm INTEGER,
    blood_pressure_systolic INTEGER
);
```

**For WO-062 (Pricing):**
```sql
CREATE TABLE subscription_tiers (
    tier_id UUID PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id),
    tier_name VARCHAR(50),
    base_price_monthly DECIMAL(10, 2),
    discounted_price_monthly DECIMAL(10, 2),
    is_data_contributor BOOLEAN
);
```
